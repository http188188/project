2.配置

使用 git config 命令来完成所有的配置

概念

  • system - 作用于整个系统
  • global - 作用于当前用户
  • local - 作用于当前项目
作用域 配置文件
system /etc/gitconfig
global ~/.gitconfig
local .git/config

基础配置

  • git config --global user.name "<user-name>"
  • git config --global user.email "<email-address>"

别名配置

  • git config --global alias.ci commit
  • git config --global alias.co checkout
  • git config --global alias.br branch
  • git config --global alias.st status
  • git config --global alias.unstage 'reset HEAD --'
  • git config --global alias.last 'log -1 HEAD'

CRLF配置

世界上最通用的纯文本编码方式是：
- UTF-8 无 BOM
- Unix 风格换行符（LF）
所以最推荐的git关于CRLF的配置为：
- core.autocrlf false
- core.safecrlf true

core.autocrlf

选项 解释
true 检出时转换为CRLF，提交时转换为LF
input 检出时不转换，提交时转换为LF
false 提交检出时均不转换

core.safecrlf

选项 解释
true 拒绝提交包含混合换行符的文件
false 允许提交包含混合换行符的文件
warn 提交包含混合换行符的文件时给出警告

diff和merge配置

  • git config --global core.editor vim|emacs 默认调用的编辑器
  • git config --global diff.tool p4merge 默认使用的外部diff工具
  • git config --global difftool.p4merge.cmd <path-to-p4merge> \$LOCAL \$REMOTE difftool使用的外部工具路径
  • git config --global merge.tool p4merge 默认使用的外部merge工具
  • git config --global mergetool.p4merge.cmd <path-to-p4merge> \$BASE \$LOCAL \$REMOTE \$MERGED mergetool使用的外部工具路径

用户名密码配置

  • git config --global credential.helper store 这将会在 ~/.gitcredentials 文件中保存每一个访问过的库的用户名和密码（明文）

3.基础使用

  1. git init
  2. git clone
  3. git add
    • 将一个文件添加到git的track中
    • 将一个已经track的文件添加到暂存区
    • 将一个冲突的文件标记为冲突解决
    • git add -p 交互式的添加修改到暂存区
  4. git status
  5. git diff
    • git diff 查看工作区到暂存区的修改
    • git diff --cached | --staged 暂存区到最新的提交的修改
  6. git commit
    • git commit -a 在执行提交前运行 git add -A
    • git commit --amend 反悔上一次的提交
  7. git reset
  8. git rm
    • git rm 在文件系统中删除文件同时在git中删除文件
    • git rm --cached 在git中删除文件的同时保留文件系统中的文件
  9. git mv
  10. git branch
    • git branch --track 设置本地分支追踪远端分支
  11. git checkouot
    • git checkout -b 直接新建一个分支然后切换到该分支
  12. git merge
    • git merge --no-ff 强制产生merge-commit而不是默认的fast-forward
  13. git rebase
    • git rebase -i 可以压缩多个commit为一个commit
  14. git push
    • git push -u 建立本次推送的本地分支和远端分支间的track
  15. git pull
  16. git fetch
  17. git tag
    • git tag -a 建立带附注的标签
  18. git stash
  19. git blame
